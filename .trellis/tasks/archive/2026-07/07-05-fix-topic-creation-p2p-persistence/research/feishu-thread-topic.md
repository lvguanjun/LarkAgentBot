# 飞书话题资料摘要

## 来源

- 用户提供的飞书开放平台话题文档摘录，已在本文件中保留任务所需结论；本文件不依赖临时源文件继续存在。

## 话题概念

- 话题是飞书消息的一种展示和讨论形式，用于把围绕同一主题的讨论聚合在一个回复串中。
- 相比普通消息平铺展示，话题用于聚焦讨论，并减少不关心该主题成员收到的打扰。
- 每个话题有租户内唯一 ID：`thread_id`。
- `thread_id` 格式示例为 `omt_...`。

## thread_id 获取方式

- 在话题形式群中，调用发送消息、回复消息、转发消息等接口，可从响应结果获取 `thread_id`。
- 在普通消息形式群中，调用回复消息接口，并传入 `reply_in_thread=true`，可将某一消息以话题形式回复，并从响应结果获取 `thread_id`。
- 监听接收消息事件时，如果消息属于话题，可从事件体获得 `thread_id`。
- 调用获取指定消息内容或获取会话历史消息接口时，如果消息属于话题，可从响应体获取 `thread_id`。

## 相关 API 语义

- “以话题的方式进行回复”指调用回复消息接口时填写 `reply_in_thread=true`。
- 查询话题历史消息时，使用 `container_id_type=thread`，`container_id` 填写 `thread_id`。
- 获取话题模式群中的所有消息时，可先按 `container_id_type=chat` 查询群聊根消息，再对存在 `thread_id` 的根消息按 `container_id_type=thread` 查询话题内消息。

## 任务相关结论

- 普通消息形式群中，要把某条消息创建为话题，需要回复该消息并设置 `reply_in_thread=true`。
- 创建或进入话题后，后续会话隔离应围绕 `thread_id`。
